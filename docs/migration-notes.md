# Migration Notes

This kit intentionally avoids IDE-specific command surfaces.

## What to keep from other rule systems
- solver loop
- root-cause debugging
- read-before-edit discipline
- proof-based closeout
- explicit status language
- context compression discipline

## What not to copy blindly
- IDE-specific command names
- model branding and benchmark marketing
- runtime assumptions from a different tool stack
- duplicated policy spread across too many always-on files

## Hermes-native adaptation principle
Keep the operating law.
Swap the tool mapping.
Remove the branding.
Tighten the verification.
