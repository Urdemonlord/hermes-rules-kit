# v0.2.0 Release Notes

## Summary

v0.2.0 adds a durable multi-worker operating model to Hermes Rules Kit.

The repo now has a documented answer to a question that was previously implicit:
**when should a serious Hermes workflow stay single-agent, when should it use short-lived subagents, and when should it escalate into Kanban-backed swarm orchestration?**

This release also adds a first-pass Meow worker pack so swarm behavior is packaged as modular skills rather than buried in prose.

## Highlights

### 1. Swarm mode is now explicitly defined
The repo now includes `docs/swarm-mode.md`, which defines:
- Direct Mode
- Strike Team Mode
- Swarm Mode

It also explains:
- when swarm overhead is justified
- why Kanban is the durable spine for long-horizon work
- why verification must stay role-separated
- which anti-patterns make a swarm worse than a single agent

### 2. New swarm overlay for repo-local adoption
A new `templates/swarm-AGENTS.overlay.md` lets adopters add swarm policy only where it is useful.

This matters because swarm logic should not bloat the always-on core in repos that only need direct execution discipline.

### 3. New Meow swarm skill
The repo now includes `skills/meow-swarm-mode/SKILL.md`.

This skill packages:
- mode selection policy
- escalation heuristics
- role separation
- orchestration pattern
- closeout expectations

The goal is to make swarm behavior loadable on demand instead of pretending every task should always run through a board.

### 4. First-pass worker skill pack
This release adds a baseline set of worker-role skills:
- `meow-orchestrator`
- `meow-researcher`
- `meow-builder`
- `meow-verifier`
- `meow-reviewer`

These are intentionally narrow.
They are meant to create clear ownership boundaries, not a theatrical cast of overlapping agents.

### 5. Example Kanban workflow
The repo now includes a worked example showing how a Meow swarm can be represented as a Kanban task graph with:
- task creation
- assignment by role
- dependency links
- handoff comments
- verification gate
- final closeout

## Why this release matters

Before v0.2.0, the repo had strong discipline for proof, verification, and tool-grounded execution, but its multi-worker story remained under-specified.

That left a gap between:
- short-lived `delegate_task` usage
- and durable, board-backed multi-agent execution

v0.2.0 closes that gap with an explicit policy and a reusable packaging surface.

## Upgrade guidance

If you already use the kit:
1. keep the base AGENTS/core law unchanged unless you need orchestration
2. add `templates/swarm-AGENTS.overlay.md` only in repos where durable multi-worker execution is common
3. load `skills/meow-swarm-mode/` only for tasks that actually benefit from handoff and verifier separation
4. use `examples/kanban-swarm-workflow.md` as the canonical starting point for role-based Kanban execution

## Known limitations
- The worker pack is still documentation-first; it is not yet wired to automated board bootstrapping
- Swarm mode remains a policy layer, not a one-command runtime feature
- More real-world Kanban examples are still needed across research, coding, and ops workflows
