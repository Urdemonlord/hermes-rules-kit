# Failure Modes

These examples exist to make the rules feel earned rather than aspirational.

Each file shows:
- the symptom
- the bad agent behavior
- why it failed
- which rule should have prevented it
- what the corrected behavior looks like

Current examples:
- `unverified-fixed-claim.md`
- `stale-plan-after-tool-result.md`
- `symptom-patch-instead-of-invariant-owner.md`
- `service-claimed-up-without-health-check.md`
- `context-contamination-across-projects.md`

Start here if you want to understand why the repo emphasizes proof, stale-plan detection, invariant ownership, direct health verification, and strict context isolation.
